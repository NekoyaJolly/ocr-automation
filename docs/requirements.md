# OCR Automation App — 要件定義書

## 1. プロジェクト概要

### 1.1 目的
スキャン画像・撮影画像から手書き文字を OCR で読み取り、ユーザーが定義したテンプレートに沿ったフォーマットで自動的に出力し、さらに自動印刷までを一気通貫で実行するデスクトップアプリケーション。

### 1.2 対象ユーザー
- 主に日本語(漢字・かな・英数字混在)の手書き文書を扱うユーザー
- Python やコマンドライン操作の知識を持たない一般エンドユーザー
- 個人利用および小規模配布(数人〜数十人規模)を想定

### 1.3 動作環境
- Windows 10 以降 (64bit)
- macOS 12 (Monterey) 以降
- インターネット接続不要(完全オフライン動作)
- GPU 不要(CPU のみで動作)

## 2. 機能要件

### 2.1 コア機能

#### F-01: 入力フォルダ監視(サブフォルダベース運用)
- ユーザーが指定した「入力ルートフォルダ」配下のサブフォルダを再帰的に監視
- 各サブフォルダはテンプレートセットと 1:1 で紐付けられる
  - 例: `入力/納品書/` → 納品書セット、`入力/領収書/` → 領収書セット
- サブフォルダとテンプレートセットの対応は GUI で設定し、設定ファイルに永続化
- 新規ファイル(対応画像形式: jpg, jpeg, png, tiff, tif, jp2, bmp)が追加されたことを検知
- ファイル書き込み完了を待ってからトリガー(書き込み中のファイルを誤って処理しない)
- 監視は GUI 起動中バックグラウンドで継続
- 紐付けされていないサブフォルダに置かれたファイルは無視(ログに INFO 記録)

#### F-02: OCR 処理
- 検知した画像ファイルを NDL OCR Lite で OCR 処理
- 処理は内部的に同一プロセス内で実行(subprocess ではなく Python モジュールとして直接 import)
- 処理結果として構造化データ(テキスト + 位置情報 + 信頼度など)を取得
- エラー発生時はログに記録し、対象ファイルは「失敗」フォルダへ移動

#### F-03: テンプレートセット適用
- 1 つの OCR 結果に対して、複数のテンプレートを同時に適用可能
- テンプレートセット = 複数のテンプレート + 各テンプレートの有効/無効 + 各テンプレート個別の出力先・印刷設定
- 各テンプレートは独立した「ソース項目名 → 出力項目名」のフィールドマッピングを定義
- 出力フォーマットはテンプレートごとに選択可能: txt / docx / xlsx / pdf
- **部分成功を許容**: セット内の一部テンプレート出力が失敗しても、成功した分は出力する
- 単一テンプレートのみのセット(従来の 1:1 ケース)も同じ仕組みで扱える

#### F-04: 出力フォルダ書き出し
- テンプレート適用後のファイルをユーザー指定の出力フォルダへ保存
- セット内の各テンプレートは個別のサブフォルダへ出力可能(例: `出力/invoices/`, `出力/receipts/`)
- ファイル名は命名規則(タイムスタンプ・元ファイル名・テンプレート名などの組み合わせ)に従う

#### F-04b: 失敗時の自動リトライ
- セット内のテンプレート適用または出力で失敗が発生した場合、失敗した**テンプレート単位**でリトライ
- リトライ上限: **2 回**(初回 + リトライ 2 回 = 最大 3 回試行)
- リトライ間隔: 指数バックオフ(1秒 → 3秒)
- 全リトライ失敗後、対象ファイルと失敗情報を「失敗フォルダ」へ移動
- 部分成功(一部テンプレートのみ失敗)の場合、成功分は通常出力フォルダ・失敗分のみ失敗フォルダへ記録

#### F-05: 自動印刷(テンプレート単位制御)
- 出力フォルダに新規ファイルが追加されたことを検知し、自動的にプリンタへ送信
- 印刷の有効/無効は**テンプレート単位**で制御可能
  - 例: 請求書テンプレートは印刷 ON、控えテンプレートは印刷 OFF(ファイル保存のみ)
- テンプレートセット定義内に各テンプレートの `auto_print` フラグを持つ
- 印刷先プリンタはユーザーが事前に選択(テンプレート単位で別プリンタも指定可能)
- 印刷成功/失敗のログを記録

### 2.2 GUI 機能

#### F-10: メインウィンドウ
- 監視状態(動作中/停止中)を視覚的に表示
- リアルタイム処理ログを表示(直近 100 件程度をスクロール表示)
- 開始/停止ボタン
- 設定画面への遷移ボタン

#### F-11: フォルダ設定画面
- 入力ルートフォルダ・出力ルートフォルダ・失敗フォルダの3つを GUI で選択
- 入力ルート配下のサブフォルダ一覧を表示し、各サブフォルダにテンプレートセットを割り当てる UI を提供
- フォルダパスの可視化と「フォルダを開く」ボタン
- 設定は永続化(次回起動時に復元)

#### F-12: テンプレートエディタ
- テンプレートの新規作成・編集・削除・複製
- 視覚的なフィールドマッピング UI
  - 左ペイン: OCR から検出される項目(ソース)
  - 右ペイン: 出力フォーマットの項目(ターゲット)
  - ドラッグ&ドロップまたはクリック選択でマッピング
- 出力フォーマット(txt/docx/xlsx/pdf)の選択
- ファイル命名規則の定義
- テンプレートのプレビュー機能

#### F-12b: テンプレートセット管理画面
- テンプレートセットの新規作成・編集・削除・複製
- セットに含めるテンプレートの追加/削除/有効化/無効化
- 各テンプレートの個別設定: 出力サブフォルダ・自動印刷 ON/OFF・使用プリンタ
- セットのプレビュー(セット適用時にどのファイルが何個生成されるかを事前確認)

#### F-13: プリンタ設定画面
- システムに登録されているプリンタの一覧表示
- デフォルトプリンタの選択
- 自動印刷の ON/OFF 切り替え
- 印刷部数・用紙サイズなどの基本オプション

### 2.3 サブ機能

#### F-20: ログ管理
- 全ての処理(OCR・出力・印刷)をログファイルに記録
- ログレベル: INFO / WARNING / ERROR
- ログファイルのローテーション(日次)

#### F-21: 設定の永続化
- ユーザー設定は JSON 形式でローカル保存
- 保存先はユーザーホーム配下の専用ディレクトリ
  - Windows: `%APPDATA%/OCRAutomation/`
  - macOS: `~/Library/Application Support/OCRAutomation/`

#### F-22: テンプレート/テンプレートセットのインポート/エクスポート
- テンプレートファイル(YAML)とテンプレートセットファイル(YAML)を別 PC に持ち運べるよう、エクスポート/インポート機能を提供
- セットをエクスポートする際は、参照されているテンプレートも一緒に zip にまとめる

## 3. 非機能要件

### 3.1 パフォーマンス
- 1枚の A4 画像(300dpi)に対する OCR 処理時間: 30秒以内(一般的なノート PC で)
- GUI の応答性: OCR 処理中も UI がフリーズしないこと(別スレッドで実行)

### 3.2 信頼性
- OCR 処理失敗時もアプリケーションは継続動作する
- 失敗したファイルは「失敗」フォルダに退避し、ログに理由を記録
- アプリ予期せぬクラッシュ時、再起動で監視状態を自動復元

### 3.3 配布性
- **完全バンドル方式**: NDL OCR Lite 本体・モデルファイル・Python ランタイム・全依存ライブラリを単一インストーラに内包
- ユーザー側での Python インストール・依存ライブラリ導入は一切不要
- インストーラ形式: Windows は `.exe`(またはインストーラ)、macOS は `.app`(または `.dmg`)
- 配布物のサイズは 1〜2GB を許容範囲とする

### 3.4 拡張性
- OCR エンジンは抽象インターフェース経由で呼び出し、将来的に別エンジン(TrOCR 等)へ差し替え可能な構造とする
- 出力フォーマットも抽象化し、新規フォーマット(例: HTML, Markdown, JSON)の追加を容易にする

### 3.5 ライセンス
- NDL OCR Lite: CC BY 4.0(クレジット表記が必要)
- アプリ本体ライセンス: 未定(MIT または Apache 2.0 を想定)
- 「Powered by NDL OCR Lite」のクレジットを About 画面に表示

## 4. 技術スタック(確定)

### 4.1 言語・ランタイム
- Python 3.11

### 4.2 主要ライブラリ
| 用途 | ライブラリ |
|------|-----------|
| GUI | PySide6 (Qt6) |
| フォルダ監視 | watchdog |
| OCR エンジン | NDL OCR Lite (内部 import) |
| 設定管理 | pydantic + JSON |
| テンプレート定義 | PyYAML |
| Word 出力 | python-docx |
| Excel 出力 | openpyxl |
| PDF 出力 | reportlab |
| ログ | 標準 logging + RotatingFileHandler |
| 印刷(Win) | pywin32 |
| 印刷(Mac) | subprocess + lp コマンド |

### 4.3 ビルド・配布
- パッケージング: PyInstaller(`--onedir` 形式)
- CI ビルド: GitHub Actions(Windows / macOS マトリクス)
- バージョン管理: Git + GitHub
- 依存管理: uv(または pip + requirements.txt)

## 5. 制約事項

### 5.1 NDL OCR Lite に関する制約
- 手書き文字対応は公式に「実験的・試行的」と位置付けられている
- 精度は資料の種類により差がある
- ライセンスは CC BY 4.0 のため、再配布時はクレジット表記必須

### 5.2 配布上の制約
- 単一プロセス内 import 方式のため、NDL OCR Lite の API 構造に依存する
- NDL OCR Lite のアップデート時は、開発者が手動で取り込む必要あり

### 5.3 機能スコープ外
- クラウド連携(意図的に除外、オフライン優先)
- 複数 PC 間の設定同期
- ユーザー認証・マルチユーザー対応
- OCR 結果の手動補正 UI(初版では対応しない)

## 6. 開発フェーズ

### Phase 1: コア機能プロトタイプ(GUI なし)
- watchdog でフォルダ監視
- NDL OCR Lite を import で呼び出し
- 単純なテキスト出力
- 設定はハードコード

### Phase 2: GUI 実装
- PySide6 でメインウィンドウ・設定画面・ログ表示
- フォルダ設定の永続化

### Phase 3: テンプレートエンジン + テンプレートセット
- YAML テンプレート読み込み
- フィールドマッピング適用
- 複数出力フォーマット対応
- **テンプレートセット機構**: 1 OCR 結果に複数テンプレートを適用
- **失敗時リトライ機構**: テンプレート単位で最大 2 回リトライ
- **サブフォルダ → セット紐付け**: 入力サブフォルダごとに使用セットを切替

### Phase 4: テンプレート/セットエディタ GUI
- 視覚的なフィールドマッピング編集 UI
- テンプレートセット管理画面(セットへのテンプレート追加・印刷フラグ設定)

### Phase 5: 印刷機能
- クロスプラットフォーム印刷抽象化
- プリンタ選択 UI

### Phase 6: パッケージング
- PyInstaller でのビルド
- GitHub Actions CI 構築
- インストーラ生成

### Phase 7: テスト・配布準備
- 各 OS での動作確認
- ドキュメント整備
- 初回リリース
